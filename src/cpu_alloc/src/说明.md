## PCIe 地址如何区分 32 个模块

整个地址路径是 **三级解码**：

### 第一级：PCIe BAR → 本地总线地址

`pcie_regif_gen3` 模块解析 PC 通过 BAR0 发来的 CQ TLP（寄存器读写请求），从中提取出 **19bit 的 `r_addr[18:0]`**（字节地址）以及读写使能信号。

### 第二级：`cpu_alloc_32users` — 地址分块（核心！）

在 [BHD_C720_top.v L585-602](file:///c:/Users/18197/Desktop/code/c720/src/top/BHD_C720_top.v#L585-L602) 中：

```verilog
.cpu_addr  (r_addr[14:2])    // 取 bit[14:2] → 13bit 地址
```

**为什么取 `[14:2]`？**
- `r_addr` 是字节地址，bit[1:0] 是字内偏移（32bit 寄存器对齐），所以丢弃
- bit[14:2] = **13bit 字地址**，范围 0x0000 ~ 0x1FFF

### CPU_BOUNDARY 参数定义了 32 块的划分

在 [cpu_alloc_32users.v L27](file:///c:/Users/18197/Desktop/code/c720/src/cpu_alloc/src/cpu_alloc_32users.v#L27) 中：

```
CPU_BOUNDARY = {
  {13'h1fff, 13'h1f00},  // block 31: 0x1F00 ~ 0x1FFF
  {13'h1eff, 13'h1e00},  // block 30: 0x1E00 ~ 0x1EFF
  ...
  {13'h01ff, 13'h0100},  // block  1: 0x0100 ~ 0x01FF
  {13'h00ff, 13'h0000}   // block  0: 0x0000 ~ 0x00FF
}
```

**每块固定大小 = 0x100 个字地址 = 256 个寄存器（32bit 宽）= 1KB 字节空间**

### 第三级：`cpu_process` — 地址比较 + 片选生成

[cpu_process.v L54](file:///c:/Users/18197/Desktop/code/c720/src/cpu_alloc/src/cpu_process.v#L54) 中的核心逻辑：

```verilog
assign cpu_cs_curchip = (cpu_addr >= CPU_BOUNDARY[0+:CPU_ADDR_W] && 
                         cpu_addr <= CPU_BOUNDARY[CPU_ADDR_W+:CPU_ADDR_W]) 
                        ? cpu_cs : 1'b1;
```

**原理**: 将输入的 13bit 地址与自己的上下界做 **范围比较**：
- 地址落在范围内 → `cpu_u_cs` 传递有效的 `cpu_cs`（低有效）
- 地址不在范围内 → `cpu_u_cs = 1`（不选中）

同时 `cpu_u_addr` 直接透传全 13bit 地址（**不做偏移减法**），各子模块内部只取低 8bit（`[7:0]`）作为寄存器地址。

### 图示

```
PC 上位机写: BAR0 + 0x0C34  (字节地址)
                    │
      pcie_regif_gen3 解析 CQ TLP
                    │
              r_addr = 0x0C34 (19bit)
                    │
         取 r_addr[14:2] = 0x30D (13bit 字地址)
                    │

                    │

         │  block3 的 cpu_u_cs=0  │  其余 31 块 cpu_u_cs=1

                    │

```

**总结**：核心机制就是 `cpu_alloc_32users` 用 **13bit 地址的高 5bit**（bit[12:8]）作为 block 编号（0~31），低 8bit（bit[7:0]）作为块内寄存器偏移。每个 `cpu_process` 实例通过 **地址范围比较** 生成各自的片选信号 `cpu_u_cs`，只有地址匹配的那一个块会被选中，其余均为不选中状态。
