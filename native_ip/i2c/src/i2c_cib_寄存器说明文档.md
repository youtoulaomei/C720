# I2C CIB 寄存器功能说明与排查手段

> 源文件: `i2c_cib.v`  
> 模块名: `i2c_cib`  
> 功能概述: I2C 主控制器的配置接口模块，提供 I2C 物理层参数配置、传输控制和状态监控

---

## 一、寄存器总览

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x00 | I2C_00 | R | 版本号 (VERSION) |
| 0x01 | I2C_01 | R | 编译日期 {YEAR, MONTH, DAY} |
| 0x02 | I2C_02 | R/W | 测试寄存器 (写入取反存储) |
| 0x04 | I2C_04 | R/W | I2C 启动控制: bit[0]=i2c_start |
| 0x05 | I2C_05 | R/W | SCL 时钟周期: bit[23:0]=i2c_scl_prd，复位值=0x3E8 (100KHz) |
| 0x06 | I2C_06 | R/W | 操作参数: bit[16]=last_read_ack, bit[13:8]=i2c_op_len, bit[2:0]=i2c_header_len |
| 0x07 | I2C_07 | R/W | 时序参数: bit[31:16]=i2c_sto_setup, bit[15:0]=i2c_sta_hold |
| 0x08 | I2C_08 | R | 历史告警: bit[0]=his_slave_no_ack（从机无应答历史） |
| 0x09 | I2C_09 | R | 当前状态: bit[0]=cur_i2c_master_free（主机空闲状态） |

---

## 二、寄存器详细说明

### 2.1 I2C_00 (0x00) - 版本号
- **类型**: 只读
- **位域**: [31:0] = VERSION 参数（默认 0x00_00_00_01）
- **作用**: 标识 I2C CIB 模块版本

### 2.2 I2C_01 (0x01) - 编译日期
- **类型**: 只读
- **位域**: [31:16]=YEAR, [15:8]=MONTH, [7:0]=DAY
- **默认**: 2017年12月12日

### 2.3 I2C_02 (0x02) - 测试寄存器
- **类型**: 读写（写入时取反存储！）
- **作用**: 总线读写回环测试
- **注意**: 写入 D 后读回 ~D，用于验证总线完整性
- **排查用途**: 写入 0x5A5A5A5A，应读回 0xA5A5A5A5

### 2.4 I2C_04 (0x04) - I2C 启动控制
- **类型**: 读写
- **位域**: bit[0] = `i2c_start`
- **作用**: 写1触发 I2C 主机开始一次传输
- **排查用途**: 确认是否正确下发启动命令

### 2.5 I2C_05 (0x05) - SCL 时钟周期
- **类型**: 读写
- **复位值**: 0x000003E8 (1000)
- **位域**: bit[23:0] = `i2c_scl_prd`
- **作用**: 设置 I2C SCL 时钟周期，频率 = sys_clk / i2c_scl_prd
  - 默认值 1000 → 100MHz/1000 = 100KHz（标准模式）
  - 最低频率约 10KHz
- **排查用途**: 
  - I2C 通信不稳定时增大此值降低频率
  - 确认频率在从设备支持范围内（标准100K / 快速400K）

### 2.6 I2C_06 (0x06) - 操作参数
- **类型**: 读写
- **复位值**: 0x00000022
- **位域**:
  - bit[16]: `last_read_ack` — 最后一字节读ACK控制（0=NACK, 1=ACK）
  - bit[13:8]: `i2c_op_len` — 操作数据长度（字节数，最大63字节）
  - bit[2:0]: `i2c_header_len` — 头部长度（设备地址+寄存器地址字节数）
- **默认**: header_len=2, op_len=2
- **排查用途**: 
  - 确认头部长度与设备地址格式匹配（7bit 地址=1字节头部，10bit 地址=2字节头部）
  - 确认操作长度与实际读写数据量一致

### 2.7 I2C_07 (0x07) - 时序参数
- **类型**: 读写
- **复位值**: 0x00FF00FF
- **位域**:
  - bit[31:16]: `i2c_sto_setup` — STOP 建立时间（时钟周期数）
  - bit[15:0]: `i2c_sta_hold` — START 保持时间（SDA 高到低跳变后的保持时间）
- **作用**: 配置 I2C 起始和停止条件的时序参数
- **排查用途**: 
  - I2C 总线时序不满足规范要求时调整此值
  - 标准模式下 START hold ≥ 4.0μs, STOP setup ≥ 4.0μs

### 2.8 I2C_08 (0x08) - 从机无应答告警（历史）
- **类型**: 只读（读清）
- **位域**: bit[0] = `his_slave_no_ack`
- **作用**: 记录是否发生过从机无应答（NACK）事件
- **清除方式**: 读取本寄存器时自动清除（alarm_history 机制）
- **排查用途**: 
  - 若此位为1，说明从设备未正确响应
  - 检查从设备地址是否正确、从设备是否上电、I2C 总线是否有上拉电阻

### 2.9 I2C_09 (0x09) - 主机空闲状态
- **类型**: 只读
- **位域**: bit[0] = `cur_i2c_master_free`
- **作用**: 指示 I2C 主控制器是否空闲（1=空闲，0=忙）
- **排查用途**: 
  - 发起新传输前需确认此位为1
  - 若长时间为0，可能 I2C 总线被拉低或控制器卡死

## 三、问题排查手册

### 3.1 I2C 总线不通
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 验证总线 | 写 0x02=0x12345678 后读回 | 读回 0xEDCBA987 (取反) |
| 2. 检查空闲状态 | 读 0x09 | bit[0]=1 |
| 3. 检查从机地址 | 确认 I2C 头部数据 | 地址正确 |
| 4. 检查上拉电阻 | 测量 SDA/SCL 空闲电平 | 应为高电平 |

### 3.2 从机无应答 (NACK)
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读告警寄存器 | 读 0x08 | bit[0]=1 确认 NACK |
| 2. 检查从设备地址 | 确认7bit/10bit地址格式 | 地址匹配 |
| 3. 检查 header_len | 读 0x06 bit[2:0] | 与地址格式匹配 |
| 4. 降低 SCL 频率 | 增大 0x05 的值 | 通信稳定 |
| 5. 示波器抓波 | 观察 SDA/SCL 波形 | 时序正确 |

### 3.3 I2C 主机卡死
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读空闲状态 | 读 0x09 | 长时间为0则卡死 |
| 2. 检查 SDA 电平 | 测量 SDA 引脚 | 空闲应为高 |
| 3. 总线复位 | 发送9个 SCL 脉冲 | SDA 释放 |
| 4. 模块复位 | 上层 soft_rst | 状态机恢复 |

### 3.4 数据读写错误
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 检查操作长度 | 读 0x06 | op_len 与实际数据量一致 |
| 2. 检查时序参数 | 读 0x07 | START/STOP 时序满足规范 |
| 3. 检查 last_read_ack | 读 0x06 bit[16] | 最后一字节应发 NACK(0) |
| 4. 降低频率排查 | 修改 0x05 | 排除频率过高导致的问题 |
