# LVDS 7K CIB 寄存器功能说明与排查手段

> 源文件: `lvds_7k_cib.v`  
> 模块名: `lvds_7k_cib`  
> 功能概述: Kintex-7 系列 LVDS ADC 接口的配置模块，提供 ADC 通道使能、IDELAY 校准、测试图案配置、直流偏置管理

---

## 一、寄存器总览

### 基础寄存器

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x00 | LVDS_00 | R | 版本号 |
| 0x01 | LVDS_01 | R | 编译日期 |
| 0x02 | LVDS_02 | R/W | 测试寄存器，复位值 0xDEAD_BEEF |

### ADC 通道配置

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x04 | LVDS_04 | R/W | ADC 通道使能: bit[3:0]=adc_en |
| 0x05~0x07 | LVDS_05~07 | R/W | 预留配置寄存器 |
| 0x09 | LVDS_09 | R | FIFO 溢出告警(读清): his_fifo_overflow |
| 0x0A | LVDS_0A | R/W | 符号模式: sign_mod |

### IDELAY 校准寄存器

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x10 | LVDS_10 | R/W | 通道0: bit[4]=st_clr, bit[0]=dly_inc |
| 0x11 | LVDS_11 | R/W | 通道0 延迟值: bit[8:0]=dly_value，复位值=16 |
| 0x12 | LVDS_12 | R/W | 通道1: bit[4]=st_clr, bit[0]=dly_inc |
| 0x13 | LVDS_13 | R/W | 通道1 延迟值: bit[8:0]=dly_value，复位值=16 |
| 0x18 | LVDS_18 | R | 通道0 PNS 接收状态: bit[0]=pns_rcvd |
| 0x19 | LVDS_19 | R | 通道1 PNS 接收状态: bit[0]=pns_rcvd |

## 二、关键寄存器详细说明

### 2.1 LVDS_04 (0x04) - ADC 通道使能
- **类型**: 读写
- **位域**: bit[3:0] = `adc_en`
- **作用**: 使能/关闭各 ADC 数据通道
  - bit[0]: 通道0 使能
  - bit[1]: 通道1 使能
  - bit[2]: 通道2 使能
  - bit[3]: 通道3 使能

### 2.2 LVDS_09 (0x09) - FIFO 溢出告警
- **类型**: 只读（读清）
- **位域**: bit[LVDS_NUM-1:0] = `his_fifo_overflow`
- **作用**: 记录各通道 FIFO 是否发生过溢出
- **排查用途**: FIFO 溢出意味着数据丢失，需检查下游消费速度

### 2.3 IDELAY 校准寄存器 (0x10~0x13)
- **st_clr**: 写1清除统计信息，脉冲操作（写后自动清零）
- **dly_inc**: 写1增加 IDELAY 延迟值，脉冲操作
- **dly_value**: 设定的目标延迟值（0~511 taps）
- **复位值**: 16 taps
- **排查用途**: 
  - LVDS 数据采样窗口调整的核心寄存器
  - 数据错误时需重新调整延迟

### 2.4 PNS 接收状态 (0x18~0x19)
- **类型**: 只读
- **位域**: bit[0] = `pns_rcvd`
- **作用**: 指示 PN 测试序列是否被正确接收
- **排查用途**: IDELAY 校准过程中用于判断采样窗口是否正确

### 2.5 IDELAY 当前值 (0x20~0x27)
- **类型**: 只读
- **格式**: 每寄存器包含两路数据线的 IDELAY 值
  - bit[24:16]: 奇数线延迟值
  - bit[8:0]: 偶数线延迟值
- **排查用途**: 确认各数据线的实际延迟值

### 2.6 测试图案 (0x30~0x32)
- **作用**: ADC 测试模式下的期望数据图案
- **排查用途**: 与 ADC 输出对比验证数据通路

### 2.7 直流偏置 (0x50~0x53)
- **位域**:
  - bit[31]: `bias_sign` — 偏置方向（0=正，1=负）
  - bit[15:0]: `bias` — 偏置量值
- **作用**: 对 ADC 数据进行直流偏置补偿

## 三、问题排查手册

### 3.1 IDELAY 校准流程
```
1. 使能 ADC 通道: 写 0x04 = 0x0F (使能全部通道)
2. 设置初始延迟: 写 0x11 = 初始 tap 值
3. 启用测试模式: ADC 配置为 PN 序列输出
4. 扫描延迟值: 调整 0x11 的值 (0~31)
5. 读 PNS 状态: 读 0x18, 找到 pns_rcvd=1 的范围
6. 选择中心值: 设定延迟为有效范围中心
7. 退出测试模式: ADC 恢复正常工作
```

### 3.2 FIFO 溢出
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读告警 | 读 0x09 | 确认哪些通道溢出 |
| 2. 检查下游消费 | 确认数据读取速度 | 消费速度 ≥ 生产速度 |
| 3. 检查时钟频率 | 确认 ADC 采样率 | 不超过 FIFO 带宽 |
| 4. 检查通道使能 | 读 0x04 | 仅使能需要的通道 |

### 3.3 ADC 数据采样错误
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 检查 IDELAY | 读 0x20~0x27 | 延迟值在有效范围 |
| 2. 测试图案验证 | 启用测试模式 | 数据与 0x30~0x32 匹配 |
| 3. PNS 校验 | 读 0x18~0x19 | pns_rcvd=1 |
| 4. 重新校准 | 执行 IDELAY 扫描 | 找到最佳采样点 |
| 5. 检查偏置 | 读 0x50~0x53 | 偏置配置合理 |

### 3.4 IO 配置脉冲
- 0x40 (io_cfg) 是脉冲寄存器，写入后下个时钟自动清零
- 0x10 (st_clr) 和 0x12 (st_clr) 同样是脉冲操作
- 确保软件不会连续重复写入
