# Multi-Function (multif_cib) 寄存器功能说明与排查手段

> 源文件: `multif_cib.v`  
> 模块名: `multif_cib`  
> 功能概述: 多通道复用器/数据选择模块的配置接口，提供多达136路用户通道的模式配置、控制使能、超时管理和状态监控

---

## 一、寄存器总览

### 基础/状态寄存器

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x01 | MUF_01 | R/W | 测试寄存器，复位值 0xDEAD_BEEF |
| 0x02 | MUF_02 | R | 帧字节长度 (frame_byte_len, 参数常量) |
| 0x10 | MUF_10 | R | 状态: bit[2]=vld_indication, bit[1]=fifo_prog_empty, bit[0]=his_fifo_overflow |
| 0x11 | MUF_11 | R | 频带信息: bit[23:0]=band |
| 0x12 | MUF_12 | R | FIFO 空状态: fifo_empty[USER_NUM-1:0] |

### 通道控制寄存器 (0x20~0x24)

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x20 | MUF_20 | R/W | 通道控制 bit[31:0] (chn_ctrl[31:0]) |
| 0x21 | MUF_21 | R/W | 通道控制 bit[63:32] |
| 0x22 | MUF_22 | R/W | 通道控制 bit[95:64] |
| 0x23 | MUF_23 | R/W | 通道控制 bit[127:96] |
| 0x24 | MUF_24 | R/W | 通道控制 bit[135:128] |

### 输入有效标志 (0x30)

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x30 | MUF_30 | R | 各通道输入数据有效指示: indata_vld_r2 |

### 通道模式寄存器 (0x40~0x61)

| 地址范围 | 功能简述 |
|----------|----------|
| 0x40~0x61 | 每个寄存器配置4个通道的模式 (每通道3bit)，共覆盖136个通道 |

格式: 每寄存器 `{chn_mode[n+3], chn_mode[n+2], chn_mode[n+1], chn_mode[n]}`, 每个字段占 bit[7:3] 间隔

### 触发/超时/握手寄存器

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x80 | MUF_80 | R/W | 触发长度: bit[18:0]=trig_len |
| 0xC1 | MUF_C1 | R/W | 超时使能: tout_en，复位值 0xFFFFFFFF (全使能) |
| 0xD0 | MUF_D0 | R/W | 超时时间 0/1: {tout_time[1], tout_time[0]}，复位值 0x03E8_03E8 |
| 0xD1 | MUF_D1 | R/W | 超时时间 2/3，复位值 0x03E8_03E8 |
| 0xD2 | MUF_D2 | R/W | 超时时间 4/5，复位值 0x0032_0032 |
| 0xE0 | MUF_E0 | R/W | 握手输入: bit[31:16]=hin_len, bit[0]=hin_en，复位值 0x04000001 |

## 二、关键寄存器详细说明

### 2.1 MUF_10 (0x10) - 综合状态寄存器
- **位域**:
  - bit[2]: `vld_indication` — 任一通道有数据输入则为1
  - bit[1]: `fifo_prog_empty` — FIFO 将空综合标志
  - bit[0]: `his_fifo_overflow` — FIFO 溢出历史（读清）
- **排查核心**: 
  - `his_fifo_overflow=1` 表示发生过数据溢出，必须排查
  - `fifo_prog_empty=1` 表示所有 FIFO 都已接近空，数据供给可能不足

### 2.2 MUF_12 (0x12) - FIFO 空状态
- **类型**: 只读
- **位域**: bit[USER_NUM-1:0] = `fifo_empty`
- **作用**: 每个 bit 对应一个用户通道的 FIFO 空状态
- **排查用途**: 快速定位哪些通道没有数据

### 2.3 通道控制寄存器 (0x20~0x24) - chn_ctrl
- **类型**: 读写
- **总宽度**: 136 bit，分布在5个寄存器中
- **作用**: 每 bit 控制一个通道的使能/开关
- **排查用途**: 确认需要的通道已使能

### 2.4 通道模式寄存器 (0x40~0x61) - chn_mode
- **类型**: 读写
- **格式**: 每个通道 3bit 模式字段
  - 每寄存器包含 4 个通道: bit[2:0], bit[10:8], bit[18:16], bit[26:24]
- **模式值含义**: 由具体业务定义（如不同的数据处理模式）
- **排查用途**: 确认通道模式配置正确

### 2.5 触发长度 (0x80) - trig_len
- **类型**: 读写
- **位域**: bit[18:0] = `trig_len`
- **作用**: 触发模式下的数据长度配置

### 2.6 超时配置 (0xC1, 0xD0~0xD2)
- **tout_en (0xC1)**: 各通道超时检测使能（复位值全1=全使能）
- **tout_time (0xD0~0xD2)**: 各超时定时器的阈值
  - 每寄存器包含2个10bit超时值
  - 复位值: 0x3E8 (1000) 和 0x32 (50)
- **排查用途**: 数据中断时检查超时配置

### 2.7 握手输入 (0xE0) 
- **位域**:
  - bit[31:16]: `hin_len` — 握手输入长度，复位值=0x400 (1024)  
  - bit[0]: `hin_en` — 握手使能，复位值=1

## 三、问题排查手册

### 3.1 数据通道无数据
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 检查有效指示 | 读 0x10 bit[2] | vld_indication=1 |
| 2. 检查通道有效 | 读 0x30 | 对应 bit 为1 |
| 3. 检查通道使能 | 读 0x20~0x24 | 对应 bit 为1 |
| 4. 检查 FIFO | 读 0x12 | 对应 bit 为0(非空) |
| 5. 检查通道模式 | 读 0x40~0x61 | 模式配置正确 |

### 3.2 FIFO 溢出
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 确认溢出 | 读 0x10 bit[0] | his_fifo_overflow=1 |
| 2. 检查各级 FIFO | 确认1/2级和输入 FIFO | 定位溢出源 |
| 3. 检查帧长度 | 读 0x02 | 帧长配置合理 |
| 4. 检查下游消费 | 确认输出速度 | 匹配输入速率 |
| 5. 减少通道数 | 修改 0x20~0x24 | 减轻带宽压力 |

### 3.3 超时告警
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 检查超时使能 | 读 0xC1 | 确认使能通道 |
| 2. 检查超时阈值 | 读 0xD0~0xD2 | 阈值设置合理 |
| 3. 检查数据源 | 确认数据是否间歇 | 数据源正常 |
| 4. 增大超时值 | 写 0xD0~0xD2 | 容忍更大间隔 |

### 3.4 频带信息异常
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读频带信息 | 读 0x11 | band 值合理 |
| 2. 检查输入数据 | 确认数据源格式 | 频带标识正确 |
