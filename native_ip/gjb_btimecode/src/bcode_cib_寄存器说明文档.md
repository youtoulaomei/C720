# B 码时间 (bcode_cib) 寄存器功能说明与排查手段

> 源文件: `bcode_cib.v`  
> 模块名: `bcode_cib`  
> 功能概述: IRIG-B 码授时模块的配置接口，提供 B 码解码参数配置、时间信息读取、RTC 管理、B 码编码输出控制

---

## 一、寄存器总览

### 基础寄存器

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x00 | BCODE_00 | R | 版本号 |
| 0x01 | BCODE_01 | R | 编译日期 |
| 0x02 | BCODE_02 | R/W | 测试寄存器 (写入取反) |

### 配置寄存器

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x03 | BCODE_03 | R/W | 控制: rtc_mode, rtc_rst, sec_framehead_en, tx_en, rx_en |
| 0x04 | BCODE_04 | R | 告警(读清): his_rec_p_err, his_vld_time_out, his_rx_loss_signal |
| 0x06~0x0C | BCODE_06~0C | R/W | 解码脉冲宽度阈值 (ctl/zero/one/9.5ms) |
| 0x10 | BCODE_10 | R/W | 统计重启: bit[0]=stat_restart |
| 0x16 | BCODE_16 | R/W | 纳秒偏移补偿: bit[9:0]=cfg_nanosec_offset，复位值=5 (50ns) |

### 解码统计寄存器 (0x11~0x15)

| 地址 | 功能 |
|------|------|
| 0x11 | 接收正极性计数最大值 |
| 0x12 | 接收正极性计数最小值 |
| 0x13 | 接收脉冲计数最大值 |
| 0x14 | 接收脉冲计数最小值 |
| 0x15 | 解码状态机状态: bit[7:4]=L2 state, bit[3:0]=L1 state |

### B码解码时间 (0x20~0x23)

| 地址 | 功能 |
|------|------|
| 0x20 | B 码年/天/时: {bcode_hour, bcode_day, bcode_year} |
| 0x21 | B 码分/秒: {bcode_sec, bcode_min} |
| 0x22 | 直秒数 (str_sec) |
| 0x23 | 控制功能字: {ctl_1func, ctl_0func} |

### RTC 时间输出 (0x30~0x33)

| 地址 | 功能 |
|------|------|
| 0x30 | RTC 年/天/时 |
| 0x31 | RTC 分/秒 |
| 0x32 | RTC 微秒/毫秒 |
| 0x33 | RTC 直秒数 |

### B码编码输出配置 (0x40~0x43)

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x40 | BCODE_40 | R/W | zero 脉冲宽度 (输出)，复位值=0x30D40 (2ms) |
| 0x41 | BCODE_41 | R/W | one 脉冲宽度 (输出)，复位值=0x7A120 (5ms) |
| 0x42 | BCODE_42 | R/W | ctl 脉冲宽度 (输出)，复位值=0xC3500 (8ms) |
| 0x43 | BCODE_43 | R/W | 控制功能字 (输出): {ctl_1_func, ctl_0_func} |

### 软件授时 (0x50~0x53)

| 地址 | 功能 |
|------|------|
| 0x50 | 授时触发: bit[0]=cfg_en |
| 0x51 | 授时日期: {cfg_day, cfg_year} |
| 0x52 | 授时时间: {cfg_sec, cfg_min, cfg_hour} |
| 0x53 | 授时亚秒: {cfg_microsec, cfg_msec} |

### 频率测量 (0x60)

| 地址 | 功能 |
|------|------|
| 0x60 | 秒脉冲频率计数: sfh_pulse_freq |

## 二、关键寄存器详细说明

### 2.1 BCODE_03 (0x03) - 主控制寄存器
- **复位值**: 0x00001000 (rtc_mode=1)
- **位域**:
  - bit[13]: `rtc_mode` — RTC 时间源（0=内部自走，1=外部B码同步）
  - bit[8]: `rtc_rst` — RTC 复位
  - bit[4]: `sec_framehead_en` — 秒帧头使能
  - bit[1]: `tx_en` — B 码发送使能
  - bit[0]: `rx_en` — B 码接收使能

### 2.2 BCODE_04 (0x04) - 接收告警寄存器
- **类型**: 只读（读清）
- **位域**:
  - bit[2]: `his_rec_p_err` — 接收极性错误历史
  - bit[1]: `his_vld_time_out` — 有效时间超时历史
  - bit[0]: `his_rx_loss_signal` — 接收信号丢失历史
- **排查用途**: 诊断 B 码接收链路问题

### 2.3 BCODE_06~0x0C - 解码脉冲宽度阈值
- **作用**: 配置 B 码解码器判别 "0"/"1"/控制脉冲的时间窗口
- **位域均为 bit[19:0]**，单位为系统时钟周期（100MHz 下 1 tick = 10ns）
- **复位值**:
  - 0x06: ctl_neg_cnt_low = 0x0C0000 → ~7.99984ms
  - 0x07: ctl_neg_cnt_high = 0x0C4100 → ~8.00016ms
  - 0x08: zero_neg_cnt_low = 0x02ED30 → ~1.99984ms
  - 0x09: zero_neg_cnt_high = 0x031D50 → ~2.00016ms
  - 0x0A: one_neg_cnt_low = 0x079110 → ~4.99984ms
  - 0x0B: one_neg_cnt_high = 0x07B130 → ~5.00016ms
  - 0x0C: ms9_5_cnt = 0x0E7EF0 → 9.5ms (秒帧头检测)

### 2.4 BCODE_15 (0x15) - 解码状态机
- **位域**:
  - bit[7:4]: `cur_l2_state` — 第二级状态机状态
  - bit[3:0]: `cur_l1_state` — 第一级状态机状态
- **排查用途**: 确认解码器状态机是否正常推进

## 三、问题排查手册

### 3.1 B 码接收信号丢失
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读告警 | 读 0x04 | bit[0]=1 确认丢信号 |
| 2. 检查输入信号 | 示波器测 B 码输入 | 应有 AM 调制波形 |
| 3. 检查接收使能 | 读 0x03 bit[0] | rx_en 应为1 |
| 4. 检查硬件连接 | 确认 B 码接收板 | 连接正常 |

### 3.2 B 码解码时间不正确
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读解码时间 | 读 0x20~0x21 | 与标准时间对比 |
| 2. 检查脉冲统计 | 读 0x11~0x14 | 脉宽在正常范围 |
| 3. 检查解码阈值 | 读 0x06~0x0C | 阈值与信号匹配 |
| 4. 检查状态机 | 读 0x15 | 状态在正常推进 |
| 5. 重启统计 | 写 0x10=0x01 | 清除统计重新采集 |

### 3.3 RTC 时间不同步
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 检查 RTC 模式 | 读 0x03 bit[13] | 1=外部同步模式 |
| 2. 读 RTC 时间 | 读 0x30~0x32 | 应与 B 码时间一致 |
| 3. 纳秒补偿 | 读 0x16 | 偏移值合理 |
| 4. 频率校验 | 读 0x60 | sfh_pulse_freq ≈ 系统时钟频率 |
| 5. 软件授时 | 写 0x51~0x53 后写 0x50=1 | 手动校时 |

### 3.4 B 码发送异常
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 检查发送使能 | 读 0x03 bit[1] | tx_en=1 |
| 2. 检查脉宽配置 | 读 0x40~0x42 | zero=2ms, one=5ms, ctl=8ms |
| 3. 检查控制功能字 | 读 0x43 | 配置正确 |
| 4. 示波器检查 | 测量 B 码输出波形 | 各脉冲宽度正确 |
