# SPI CIB 寄存器功能说明与排查手段

> 源文件: `spi_cib.v`  
> 模块名: `spi_cib`  
> 功能概述: SPI 控制器的配置接口模块，提供 SPI 物理层参数配置、收发数据缓存管理

---

## 一、寄存器总览

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x00 | CIB_00 | R | 版本号 (VERSION) |
| 0x01 | CIB_01 | R | 编译日期 {YEAR, MONTH, DAY} |
| 0x02 | CIB_02 | R/W | 测试寄存器 (test_reg)，复位值 0xDEAD_BEEF |
| 0x03 | CIB_03 | R/W | SPI 模式配置: bit[16]=spi_3line_en, bit[0]=spi_phy_rw_bit |
| 0x04 | CIB_04 | R/W | SPI 启动控制: bit[0]=spi_phy_start |
| 0x05 | CIB_05 | R/W | 数据长度配置: bit[14:8]=read_data_len, bit[6:0]=send_data_len |
| 0x06 | CIB_06 | R/W | SPI 时钟半周期: bit[15:0]=half_period，复位值=25 |
| 0x08 | CIB_08 | R/W | 等待时间: bit[15:0]=wait_time，复位值=50 |
| 0x09 | CIB_09 | R/W | 读等待时间: bit[15:0]=read_waitTime，复位值=100 |
| 0x0A | CIB_0A | R | SPI 物理层空闲状态: bit[0]=spi_phy_idle |
| 0x0B | CIB_0B | R/W | SPI 片选/读选择: bit[1]=spi_phy_ss_n_sel, bit[0]=spi_phy_rd_sel |
| 0x40~0x7F | — | R/W | TX RAM 区域（发送数据缓存，每地址存 8bit） |
| 0x80~0xBF | — | R | RX RAM 区域（接收数据缓存，每地址存 8bit） |

## 二、寄存器详细说明

### 2.1 CIB_00 (0x00) - 版本号
- **类型**: 只读
- **位域**: [31:0] = VERSION 参数
- **作用**: 标识当前 SPI CIB 模块的版本号
- **排查用途**: 确认加载的 FPGA 逻辑版本是否正确

### 2.2 CIB_01 (0x01) - 编译日期
- **类型**: 只读
- **位域**: [31:16]=YEAR, [15:8]=MONTH, [7:0]=DAY
- **作用**: 标识编译日期
- **排查用途**: 确认 FPGA 位流的编译时间

### 2.3 CIB_02 (0x02) - 测试寄存器
- **类型**: 读写
- **复位值**: 0xDEAD_BEEF
- **作用**: CPU 总线读写回环测试
- **排查用途**: 
  - 写入任意值后读回，验证 CPU 本地总线通路是否正常
  - 若读回值与写入值不一致，说明 CPU 到 SPI CIB 的总线存在问题

### 2.4 CIB_03 (0x03) - SPI 模式配置
- **类型**: 读写
- **位域**:
  - bit[16]: `spi_3line_en` — 三线 SPI 模式使能（1=三线模式，0=四线标准模式）
  - bit[0]: `spi_phy_rw_bit` — 读写方向（1=读操作，0=写操作）
- **作用**: 配置 SPI 接口的工作模式和传输方向
- **排查用途**: 确认是否选择了正确的 SPI 工作模式

### 2.5 CIB_04 (0x04) - SPI 启动控制
- **类型**: 读写
- **位域**: bit[0] = `spi_phy_start`
- **作用**: 写1触发 SPI 物理层开始一次传输
- **排查用途**: 
  - 确认软件是否正确下发了启动信号
  - 如果 SPI 无法启动，检查此寄存器是否被正确写入

### 2.6 CIB_05 (0x05) - 数据长度配置
- **类型**: 读写
- **位域**:
  - bit[14:8]: `read_data_len` — 读取数据字节长度（0~127）
  - bit[6:0]: `send_data_len` — 发送数据字节长度（0~127）
- **作用**: 配置一次 SPI 传输中发送和接收的数据长度
- **排查用途**: 
  - 若数据收发不完整，检查长度配置是否与预期一致
  - 注意发送和接收的长度需分别配置

### 2.7 CIB_06 (0x06) - SPI 时钟半周期
- **类型**: 读写
- **复位值**: 25（对应 100MHz 系统时钟下 SPI_CLK = 2MHz）
- **位域**: bit[15:0] = `half_period`
- **作用**: 设置 SPI 时钟频率，SPI_CLK = sys_clk / (2 × half_period)
- **排查用途**: 
  - SPI 通信不稳定时，适当增大此值降低 SPI 时钟频率
  - 确认 SPI 频率是否在从设备支持的范围内

### 2.8 CIB_08 (0x08) - 等待时间
- **类型**: 读写
- **复位值**: 50
- **位域**: bit[15:0] = `wait_time`
- **作用**: SPI 传输过程中的等待时间配置（以系统时钟周期为单位）
- **排查用途**: 如果 SPI 从设备响应慢，需增大此值

### 2.9 CIB_09 (0x09) - 读等待时间
- **类型**: 读写
- **复位值**: 100
- **位域**: bit[15:0] = `read_waitTime`
- **作用**: SPI 读操作等待时间配置
- **排查用途**: 读操作失败时，增大此值给从设备更多响应时间

### 2.10 CIB_0A (0x0A) - SPI 空闲状态
- **类型**: 只读
- **位域**: bit[0] = `spi_phy_idle`
- **作用**: 指示 SPI 物理层是否处于空闲状态（1=空闲，0=忙碌）
- **排查用途**: 
  - 发起新传输前需确认此位为1
  - 若长时间为0，说明 SPI 状态机可能卡死

### 2.11 CIB_0B (0x0B) - 片选/读选择
- **类型**: 读写
- **位域**:
  - bit[1]: `spi_phy_ss_n_sel` — 片选信号选择
  - bit[0]: `spi_phy_rd_sel` — 读数据来源选择
- **作用**: 控制 SPI 片选和读数据路径

### 2.12 TX RAM (0x40~0x7F) - 发送数据缓存
- **类型**: 读写
- **深度**: 64字节
- **作用**: CPU 将要发送的数据写入此区域，SPI 物理层从此区域读取数据发送
- **排查用途**: 
  - 读回此区域验证写入的发送数据是否正确
  - 使用 `send_ram_addr` 和 `send_ram_data` 端口与物理层交互

### 2.13 RX RAM (0x80~0xBF) - 接收数据缓存
- **类型**: 只读（CPU 侧），SPI 物理层写入
- **深度**: 64字节
- **作用**: SPI 物理层接收的数据存储到此区域，CPU 可读回
- **排查用途**: 
  - SPI 传输完成后读取此区域获取接收数据
  - 若读出全0或异常值，检查 SPI 连线和从设备状态

## 三、问题排查手册

### 3.1 CPU 总线不通
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读版本号 | 读 0x00 | 返回 VERSION 参数值 |
| 2. 读编译日期 | 读 0x01 | 返回正确的日期值 |
| 3. 读写测试寄存器 | 写 0x02 后读回 | 读回值 = 写入值 |
| 4. 检查片选 | 确认 cpu_cs 信号 | cs 拉低时才能访问 |

### 3.2 SPI 传输无法启动
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 检查空闲状态 | 读 0x0A | bit[0]=1 表示空闲 |
| 2. 检查配置 | 读 0x03, 0x05, 0x06 | 确认模式/长度/频率正确 |
| 3. 启动传输 | 写 0x04=0x01 | SPI 开始传输 |
| 4. 轮询状态 | 轮询 0x0A | bit[0] 由0变1表示完成 |

### 3.3 SPI 数据错误
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 检查 TX 数据 | 读 0x40~0x7F | 确认发送数据正确 |
| 2. 检查 RX 数据 | 读 0x80~0xBF | 确认接收数据合理 |
| 3. 降低 SPI 频率 | 增大 0x06 的值 | 通信稳定性改善 |
| 4. 增大等待时间 | 增大 0x08/0x09 | 给从设备更多响应时间 |

### 3.4 SPI 状态机卡死
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读空闲状态 | 读 0x0A | 长时间为0则可能卡死 |
| 2. 检查时钟 | 示波器测量 SPI_CLK | 确认时钟有输出 |
| 3. 复位模块 | 通过上层 soft_rst 复位 | 状态机恢复 |
