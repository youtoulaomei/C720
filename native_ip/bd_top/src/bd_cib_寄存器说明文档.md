# BD (北斗/GNSS) CIB 寄存器功能说明与排查手段

> 源文件: `bd_cib.v`  
> 模块名: `bd_cib`  
> 功能概述: 北斗/GNSS 定位授时模块的配置接口，提供 GGA/ZDA/RMC 报文数据读取、时间配置、PPS 管理

---

## 一、寄存器总览

### 控制/配置寄存器

| 地址 | 宏定义 | 读写 | 功能简述 |
|------|--------|------|----------|
| 0x03 | BD_03 | R/W | 秒脉冲模式: bit[0]=spulse_mode |
| 0x04 | BD_04 | R/W | 测试寄存器，复位值 0xDEAD_BEEF |
| 0x05 | BD_05 | R/W | 软复位: bit[0]=bd_soft_rst |
| 0x06 | BD_06 | R | 告警/状态: bit[6:4]=超时告警, bit[3:0]=有效标志 |
| 0x07 | BD_07 | R/W | 串口发送数据: bit[7:0]=send_data |
| 0x08 | BD_08 | R/W | 串口发送控制: bit[0]=send |
| 0x09 | BD_09 | R/W | RTC 时间源模式: bit[2:0]=rtc_mode |
| 0x0A | BD_0A | R/W | 时区配置: zone_cnt, east, west |
| 0x0B | BD_0B | R/W | 超时阈值: bit[19:0]=tmo_sec，复位值=60s |

### GGA 报文寄存器 (0x10~0x1A)

| 地址 | 功能 |
|------|------|
| 0x10 | GGA 有效标志 + ID |
| 0x11~0x12 | GGA 纬度 (wd_word0/1) |
| 0x13 | GGA 纬度方向 (wd_dir) |
| 0x14~0x15 | GGA 经度 (jd_word0/1) |
| 0x16 | GGA 经度方向 (jd_dir) |
| 0x17~0x18 | GGA 海拔高度 (high_word0/1) |
| 0x19 | GGA 海拔正负符号 |
| 0x1A | GGA 定位状态 + 卫星数 |

### ZDA 报文寄存器 (0x20~0x28)

| 地址 | 功能 |
|------|------|
| 0x20 | ZDA 有效标志 + ID |
| 0x21 | ZDA UTC 日期 {year, month, day} |
| 0x22 | ZDA UTC 时间 {hour, min, sec} |
| 0x28 | ZDA 时间类型 + PPS 状态 |

### RMC 报文寄存器 (0x30~0x3A)

| 地址 | 功能 |
|------|------|
| 0x30 | RMC 有效标志 + ID |
| 0x31~0x32 | RMC 纬度 |
| 0x33 | RMC 纬度方向 |
| 0x34~0x35 | RMC 经度 |
| 0x36 | RMC 经度方向 |
| 0x37 | RMC 定位状态 |
| 0x38 | RMC 地面航向 + 地面速度 |
| 0x39 | RMC UTC 日期 |
| 0x3A | RMC UTC 时间 |

### 软件授时寄存器 (0x40~0x45)

| 地址 | 功能 |
|------|------|
| 0x40 | 软件授时触发: bit[0]=soft_cfg |
| 0x41 | 软件设定日期 {year, month, day} |
| 0x42 | 软件设定时间 {hour, min, sec} |
| 0x43~0x45 | 软件设定亚秒 {ms, us, ns} |

### RTC 时间输出寄存器 (0x51~0x55)

| 地址 | 功能 |
|------|------|
| 0x51 | RTC 日期 {year, month, day} |
| 0x52 | RTC 时间 {hour, min, sec} |
| 0x53~0x55 | RTC 亚秒 {msec, microsec, nanosec} |

### UTC 偏移寄存器 (0x63~0x65)

| 地址 | 功能 |
|------|------|
| 0x63~0x65 | UTC 偏移 {u_ms, u_us, u_ns}，ns 复位值=30 |

## 二、关键寄存器详细说明

### 2.1 BD_06 (0x06) - 告警/状态寄存器
- **类型**: 只读
- **位域**:
  - bit[6]: `his_zda_overtime` — ZDA 报文超时告警（读清模式）
  - bit[5]: `his_gga_overtime` — GGA 报文超时告警
  - bit[4]: `his_rmc_overtime` — RMC 报文超时告警
  - bit[3]: `rmc_utc_vld` — RMC UTC 时间有效标志
  - bit[2]: `zda_utc_vld` — ZDA UTC 时间有效标志
  - bit[1]: `gga_vld` — GGA 定位有效标志
  - bit[0]: `pps_vld` — PPS 信号有效标志
- **特殊行为**: 读取后超时告警会被清除（读清）
- **排查用途**: 核心状态指示寄存器

### 2.2 BD_09 (0x09) - RTC 时间源模式
- **类型**: 读写
- **位域**: bit[2:0] = `rtc_mode`
  - 0: ZDA 报文授时
  - 1: RMC 报文授时
  - 2: 软件授时
- **排查用途**: 确认 RTC 使用了正确的时间源

### 2.3 BD_0A (0x0A) - 时区配置
- **类型**: 读写
- **复位值**: 0x0000_0801 (东8区，北京时间)
- **位域**:
  - bit[11:8]: `zone_cnt` — 时区偏移量
  - bit[4]: `west` — 西时区
  - bit[0]: `east` — 东时区
- **排查用途**: 确认时区配置正确

### 2.4 BD_0B (0x0B) - PPS 超时阈值
- **类型**: 读写
- **复位值**: 60 (60秒)
- **位域**: bit[19:0] = `tmo_sec`
- **作用**: PPS 信号超过此时间未出现则判定 pps_vld=0

## 三、问题排查手册

### 3.1 PPS 信号无效
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读状态 | 读 0x06 bit[0] | pps_vld=1 为有效 |
| 2. 检查超时 | 读 0x0B | 确认超时阈值合理 |
| 3. 测量 PPS | 示波器测 PPS 引脚 | 应有1Hz 脉冲 |
| 4. 天线检查 | 确认 GNSS 天线连接 | 天线正常、有信号 |

### 3.2 GGA/ZDA/RMC 报文超时
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 读告警 | 读 0x06 bit[6:4] | 确认哪个报文超时 |
| 2. 检查有效标志 | 读 0x06 bit[3:1] | 对应 vld 应为1 |
| 3. 检查串口连接 | 确认 GNSS 模块串口 | 波特率匹配 |
| 4. 读报文数据 | 读 0x10~0x3A | 确认数据不全为0 |
| 5. 检查定位 | 读 0x1A | gga_state ≠ 0 |

### 3.3 时间不准确
| 排查步骤 | 操作 | 预期结果 |
|----------|------|----------|
| 1. 检查 RTC 模式 | 读 0x09 | 确认时间源正确 |
| 2. 检查时区 | 读 0x0A | 确认东/西时区和偏移 |
| 3. 读 RTC 时间 | 读 0x51~0x55 | 与标准时间对比 |
| 4. 读 UTC 偏移 | 读 0x63~0x65 | 确认补偿值合理 |
| 5. 软件授时 | 写 0x41~0x45 后写 0x40=1 | 手动校时 |

### 3.4 秒脉冲频率异常
| 排查步骤 | 排查点 |
|----------|--------|
| 1. 检查脉冲模式 | 读 0x03 确认 spulse_mode (0=内部RTC, 1=外部PPS) |
| 2. 检查 sfh_pulse_freq | 此值应接近系统时钟频率（如100MHz ≈ 0x05F5E100） |
| 3. 检查 PPS 信号 | 外部 PPS 模式下确认 PPS 输入正常 |
